Introduction

This tool is mainly for programmers. It helps testing a regular expression before using it in some code. There's not as many features as in professional tools, since it is intended only to quickly test a short regular expression. Depending on your computer's capacities, the size of the subject to parse and the complexity of the regular expression, it may take more or less of your microprocessor's time to process the expression. Be sure to use reasonably-sized expressions and reasonable amounts of text.

Notations used in this document

  • A regular expression
  • Outdated / not accurate content
  • A URL
  • Any kind of variable / named parameter

Browser requirements / specifications

  • The regular expressions flavour is the one of Javascript 1.2, which is similar to Perl's one.
  • Javascript 1.5 is needed (because of exception handling).
  • DOM Level 2 support
  • CSS 2 compatible
  • A fast computer, since there are some intensive calculations.
    For instance, I have found Konqueror requiring several explicit confirmations to continue executing the script because it was too slow (heavy load) on a Pentium III 450 MHz with 128 MB SDRAM. Of course, there was a large amount of data. I will probably set up some kind of benchmark to have a better idea of the power needed.

Quick start

  1. Open the tool's web page or add it to your browser's sidebar by clicking on one of the above links / pictures
  2. Paste the text to parse in the top text area
  3. Type a regular expression in the middle field
  4. Validate by either clicking on the button or typing Enter
  5. The matches are shown in the bottom of the page
  6. For instance : copy and paste this page's content into the text field, and type the following pattern : /\be\w*\b/i to match all word beginning with a 'e' or a 'E'

Regular expressions syntax

The regular expression you are awaited to type in the middle field follows Perl's regular expressions syntax. That means : command/options/modifiers. Two commands are supported : "match" (m// operator) and "replace" (s/// operator).

  • To obtain the matches of a regular expression, type : m/pattern/modifiers or /pattern/modifiers,
  • To obtain the result of a substitution, type : s/pattern/replace/modifiers.
where :
  • pattern is the regular expression pattern
  • modifiers is a sequence of characters in {'g','i'}
  • replace is a substitution pattern

Menu

There is an auto-hiding menu at the top of the window that proposes different ways to open RegExpresso's window.

  • : Opens RegExpresso in Internet Explorer's search pane. If you bookmark this link, it will open in the search pane every time you call it.
  • : Adds RegExpresso as a new panel to the sidebar of Netscape/Mozilla familly browsers.
  • : Adds RegExpresso as a new panel to the sidebar of Opera. This also works with Firefox.
  • : Opens a new RegExpresso window, sized like a toolbar. The current content of the fields is lost.
  • : Opens the current RegExpresso (keeps the fields values) in the "_main" target. With Netscape/Mozilla and Internet Explorer, the "_main" target corresponds to the current browser's window. With other browsers, this should be a new blank window, except if there is already a frame/window named "_main". This is intended to be used only from the sidebar. Since this is a standard link, and since it is updated every time the form is validated, one can bookmark it, or open it using the contextual menus provided by the browser, which may bring more control.

Parameters

RegExpresso uses two kind of parameters :

  • standard form fields
  • URL parameters
While the formers can be altered by the user simply by typing some text in the web page, the latters require the user to add them directly to the URL used to call the tool. Indeed, those parameters are not frequently used, and it would have been wasting space to include them in the form. Additionally, the form parameters can be passed in the URL the same way, so one can bookmark the full URL with parameters to retrieve a given configuration of the tool.

In the following tables, the "short name" is the name to use in the URL for this parameter. For instance : http://regexpresso.berlios.de/regexpresso.html?highlight=on&stylesheet=http://toto.com/mystyle.css would lead to an instance of the tool with "highlight" checked and a custom stylesheet located at http://toto.com/mystyle.css.

Parameters accessed via the web form.
Short nameDescriptionExamples / values
subject A text area where to put the text the regular expression will parse. Any text will do it.
regex The regular expression, using the Perl syntax (Javascript restrictions applied). /\b\w*(e)/i will match all words ending with 'e' or 'E'
highlight Switch the 'highlight' mode, where matches are highligthed and printed at their normal place in the subject, so we don't loose the surrounding context. In normal mode (highlight is unchecked), the matches are displayed one after another, line by line. checked/"on" = highlight mode ; unchecked/""(empty) = normal mode
backref When using capturing parentheses in the regular expression, there may be one or more "sub-matches" for every match. This option tells whether or not to show them. checked/"on" = show back references ; unchecked/""(empty) = don't show

Parameters given in the URL only
Short nameDescriptionExamples / values
stylesheet An URL to a stylesheet of type "text/css" to add to the tool. Since this is CSS, this stylesheet can override the style attributes already defined in the default one embedded in RegExpresso. Any valid URL to a CSS stylesheet will do it.
empty In highlight mode, empty matches are replaced by the "empty set" character : ∅ (may not be correctly rendered here). This parameter can be set to change this default character. Any character or string (preferably encoded in UTF-8).

Tips

How to remove all empty matches from the view ?

The short answer : call this URL : http://regexpresso.berlios.de/regexpresso.html?empty=&stylesheet=http://regexpresso.berlios.de/noempty.css.

Now, the explanation.

  • You must first empty the filler string by setting it to an empty string in the URL calling RegExpresso. Here is a sample URL : http://regexpresso.berlios.de/regexpresso.html?empty=.
  • With most browsers, you will still have the borders printed at the place of the empty match. To remove them as well, you must provide RegExpresso a custom stylesheet with borders removed for the empty matches. Once again, this is done by using the a parameter of the URL, like : http://regexpresso.berlios.de/regexpresso.html?stylesheet=http://myhost/my.css. You can find such a stylesheet at http://regexpresso.berlios.de/noempty.css.

RegExpresso executes when loading if both regex and subject are given in the URL.

So you can use such URLs to remember or show the result of a regular expression without no user intervention at all.

Local installation

You can download the files for local use, but there are limits :

  • Opera : buttons "open in main window" and "open a new window" won't work because the URL contains parameters ("?param1=...&param2=...") that Opera cannot understand on local files ("file://" protocol).
  • Firefox : button "open a new window" won't work from the sidebar because it is loaded from the context of the main window, which will not permit to open local files due to security reasons.
  • With some other browser that may use relative URL in the case of local files, there may be security locks that prohibit opening such an URL from the sidebar, depending on the context of the current main window.
